# PHASE 8 CRITICAL FIX - ALERTS DATABASE SCHEMA MISMATCH

We have completed Phase 8 implementation but manual testing exposed a runtime failure.

## Current Error

Backend returns:

```text
sqlite3.OperationalError: no such column: alerts.title
```

Trace:

```text
SELECT alerts.id,
alerts.crime_event_id,
alerts.title,
alerts.description,
alerts.severity,
alerts.source,
alerts.status,
alerts.assigned_user_id,
alerts.metadata_payload,
alerts.created_at,
alerts.updated_at
FROM alerts
```

The SQLAlchemy Alert model contains these fields but the SQLite database table was created before these columns existed.

## Root Cause

The project uses:

```python
Base.metadata.create_all()
```

This does not modify existing tables.

It only creates missing tables.

Therefore the database schema and SQLAlchemy model are out of sync.
